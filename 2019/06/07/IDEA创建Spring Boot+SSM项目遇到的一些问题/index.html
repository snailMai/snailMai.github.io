IDEA搭建Spring Boot+SSM项目遇到的一些问题

发表于 2019-06-07

搭建过程

搭建过程网上教程很多，当时我使用的教程：
https://blog.csdn.net/tr1912/article/details/79371137

ps：由于我的IDEA是新版本，本人创建时跟教程有点不一样，创建时我没有选择依赖（也就是里边的1.3），后边可以根据修改pom.xml增加依赖,在pom.xml的标签新增下列依赖。

ps：有些看不懂博主的application.properties的配置，放上我的application.properties的来助于理解；
需要修改的为数据连接这块，根据自己的需求修改。

问题解决

1.java.sql.SQLException: The server time zone value xxx；

这个属于MySQL连接数据库报时区错误，在application.properties中的spring.datasource.url这一行后边加上?useUnicode=true&characterEncoding=UTF-8&useJDBCCompliantTimezoneShift=true&useLegacyDatetimeCode=false&serverTimezone=UTC。

eg：
spring.datasource.url=jdbc:mysql://localhost:3306/ssmspringboot?useUnicode=true&characterEncoding=UTF-8&useJDBCCompliantTimezoneShift=true&useLegacyDatetimeCode=false&serverTimezone=UTC

2.Loading class com.mysql.jdbc.Driver'. This is deprecated. The new driver class iscom.mysql.cj.jdbc.Driver’. The driver xxx；

这个错误是MySQL驱动地址变成了com.mysql.cj.jdbc.Driver。

不过这个错误只是警告，不影响程序运行（猜测为驱动程序自动注册了）

总结

按照博主搭建没什么问题，如果搭建的时候修改了包名和项目名，那么请注意按照自己的包名和项目名修改以下地方：

  1. application.properties中的别名；
  2. UserMapper.xml中的namespace；
